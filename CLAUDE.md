# CLAUDE.md — Routine Orchestrator

## What this project does
A GUI application for building and running automation routines — sequences of
audio playback, TTS announcements, timed waits, and external Python scripts.
Designed to be triggered via voice commands (e.g. "Alexa play X") by passing a
routine name as a CLI argument, which auto-loads and runs it then closes.

## Environment
- OS: Windows 11, PowerShell terminal inside VS Code
- Python: 3.12.10
- Activate venv: `venv\Scripts\activate` (PowerShell) or `source venv/Scripts/activate` (bash)
- Project path: C:\Users\mperr\My Drive\Python Programs\Setting Up Python Routines

## How to run
- Run GUI: `python Orchestrator_main_gui.py`
- Run routine from CLI (auto-run + auto-close): `python Orchestrator_main_gui.py RoutineName`
  - RoutineName matches a JSON file in the Routines/ folder (e.g. `PlaySilvaMeditation`)
- Install dependencies: `pip install -r requirements.txt`

## Stack & libraries
- Language: Python 3.12
- customtkinter   — modern themed GUI framework (main window and dialogs)
- tkinter         — standard library GUI (menus, file dialogs)
- pygame          — audio playback via mixer (44100 Hz stereo)
- pydub           — audio format conversion (M4A → MP3, etc.)
- gtts            — Google Text-to-Speech for announcements (free, no API key)
- pyttsx3         — local TTS engine (used in test_run.py diagnostic)
- threading       — routine execution runs on daemon thread to keep UI responsive
- subprocess      — runs external Python scripts

## Action types
- **Audio** — Play files or folders (Sequential/Random mode), with repeat count and
  optional duration limit. Folders use `.last_played.txt` bookmarks for sequential tracking.
- **Announcement** — TTS via gTTS, played through pygame
- **Wait** — Countdown timer with live UI updates
- **Script** — Run an external Python script via subprocess
