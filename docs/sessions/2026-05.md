## 2026-05-28 — Audio Sequence Editor: Save/Exit dirty-tracking pattern
- Replaced the old "Confirm" + "Exit (No Save)" buttons in `AudioSequenceEditor` with a Save/Exit pair driven by a `_is_dirty` flag.
  - **Save** starts disabled and gray; flips to enabled + green the moment any field changes. Saving applies pending edits to `action.data` in memory, marks the parent app dirty, refreshes the display, and clears the dirty flag (editor stays open).
  - **Exit** text toggles between "Exit" and "Exit without Saving" based on dirty state; width fixed to 160 to fit the longer label without resizing.
  - X (WM_DELETE_WINDOW) now mirrors Exit — no implicit save on close. The old `on_closing` method (which silently sync'd data on close) was removed.
- Dirty tracking wired up via `_mark_dirty` on: clear-all, add file, add folder, mode toggle, remove, and `<KeyRelease>` / `trace_add("write")` on every per-row entry (repeat, duration, device, volume, gap).
- `settings.json` recent_files reshuffled by the app at runtime (Michaels Alienware paths now leading).
- `.claude/settings.local.json` allowlist expanded with `./venv/Scripts/python.exe *` for ad-hoc tool invocations.
- Tracker `read` step of `/handoff` failed locally with SSL `CERTIFICATE_VERIFY_FAILED` against `oauth2.googleapis.com` — unrelated to sheet permissions; looks like a CA-cert/network config on this machine. Surfacing rather than retrying blindly.
