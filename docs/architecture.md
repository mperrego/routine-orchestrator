# Architecture — Routine Orchestrator

## Project Structure
```
Setting Up Python Routines/
├── Orchestrator_main_gui.py   — Main GUI app: build, edit, save, load, run routines
├── audio_engine.py            — Audio playback, TTS, format conversion, script runner
├── editors.py                 — AudioSequenceEditor dialog for configuring audio actions
├── settings.json              — Persisted state: last directories, recent files
├── test_run.py                — Diagnostic: list available pyttsx3 voices
├── test_script.py             — Test target for verifying external script execution
├── temp_announcement.mp3      — Temp file generated by gTTS during announcements
├── temp_converted.mp3         — Temp file from pydub format conversion
├── CLAUDE.md                  — Slim project context
├── docs/                      — Extracted detail (this file lives here)
└── Routines/                  — Saved routine JSON files
    ├── PlaySilvaMeditation.json
    ├── announceAudioAnnounce.json
    ├── test1.json
    └── test2.json
```

## Action Types
The four action types that can be sequenced into a routine:

- **Audio** — Play files or folders (Sequential/Random mode), with repeat count and optional duration limit. Folders use `.last_played.txt` bookmarks for sequential tracking.
- **Announcement** — TTS via gTTS, played through pygame.
- **Wait** — Countdown timer with live UI updates.
- **Script** — Run an external Python script via subprocess.

## Execution Model
- Routine execution runs on a **daemon thread** to keep the Tk UI responsive — never block the Tk main loop with audio or sleeps.
- External scripts launch via `subprocess`.
- Status bar reflects real-time progress; the title bar shows the loaded routine name + an unsaved-changes indicator.

## Persistence
- All persistence is JSON-based — `settings.json` for app state and `Routines/*.json` for routine definitions.
- No database. Folder bookmarks live in `.last_played.txt` files alongside the audio folders.

## Cast / Speaker Fallback
- Cast playback uses pychromecast for WiFi casting to Google/Nest speakers.
- At routine start, all configured devices (Cast + Bluetooth) are checked for reachability. If unreachable, the routine announces a fallback and uses the system default speaker.

## CLI Auto-Run
- `python Orchestrator_main_gui.py RoutineName` auto-loads, runs, and closes the GUI — designed for voice-trigger flows (e.g. "Alexa play X").
- The CLI command builder includes the full `python.exe` path, the routine name, and a log redirect.
